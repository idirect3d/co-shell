# FIX-285 测试用例

## UC-0001: handleLoopDetection 保存判模 exit_strategy

### 测试目标
验证 handleLoopDetection 中判模确认循环时，其返回的 ExitStrategy 被正确保存到 loopJudgeExitStrategy 字段。判模非循环时清空该字段。

### 前置条件
- Go 编译环境
- agent/loop.go 和 agent/run_stream.go 已完成 FIX-285 修改

### 测试步骤
1. 创建测试函数 `TestHandleLoopDetection_SavesExitStrategy`
2. 构造一个 Agent 实例，设置 cfg.LLM.LoopJudgeEnabled = true
3. 构造一个检测错误（LoopDetectedError）
4. 编写一个 fake loopJudge 插入点，返回 ExitStrategy="请调用 attempt_completion 来结束任务"
5. 调用 handleLoopDetection
6. 检查 agent.loopJudgeExitStrategy 是否正确保存

### 预期结果
- agent.loopJudgeExitStrategy 等于判模返回的 ExitStrategy
- agent.loopDetectSyncErr != nil

### 验证方法
- `go test -run TestHandleLoopDetection_SavesExitStrategy ./agent/`

## UC-0002: handleLoopDetection 判模非循环时清空 exit_strategy

### 测试目标
验证 handleLoopDetection 中判模非循环时正确清空 loopJudgeExitStrategy。

### 前置条件
- Go 编译环境

### 测试步骤
1. 创建测试函数 `TestHandleLoopDetection_ClearsExitStrategy`
2. 构造 Agent 实例，设置 loopJudgeExitStrategy 为一个非空字符串
3. 设置 cfg.LLM.LoopJudgeEnabled = true
4. 编写 fake judge 返回 IsLoop=false
5. 调用 handleLoopDetection
6. 检查 agent.loopJudgeExitStrategy 已被清空

### 预期结果
- agent.loopJudgeExitStrategy == ""
- agent.loopDetectSyncErr == nil

### 验证方法
- `go test -run TestHandleLoopDetection_ClearsExitStrategy ./agent/`

## UC-0003: run_stream 同步路径 feedback 使用 <task> 包裹 exit_strategy

### 测试目标
验证 RunStream 同步循环处理块中，当 loopJudgeExitStrategy 非空时，反馈消息使用 exit_strategy 并用 <task> 标签包裹。

### 前置条件
- Go 编译环境

### 测试步骤
1. 审查 agent/run_stream.go 中 `case "prompt"` 分支
2. 确认当 a.loopJudgeExitStrategy 非空时，loopFeedback 使用 exit_strategy
3. 确认当 a.loopJudgeExitStrategy 为空时，回退到通用模板
4. 确认 loopFeedback 始终以 `<task>\n%s\n</task>` 格式追加

### 预期结果
- a.loopJudgeExitStrategy 非空时，反馈使用判模模型的 exit_strategy
- a.loopJudgeExitStrategy 为空时，反馈使用通用模板
- 反馈消息被 `<task>` 标签包裹

### 验证方法
- 代码审查 + `go vet ./...` 通过
